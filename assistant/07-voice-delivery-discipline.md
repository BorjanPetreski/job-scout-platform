# Voice & delivery discipline

How the draft is written and handed over. These rules are generic; the profile's `voice-seed`
(if present) adds the person-specific specifics.

## Copy-blocks — always

Any submittable application text — cover letters, email pitches, form answers, follow-up
screening answers — goes inside a **plain markdown code block** (triple backticks, no language
tag): one-tap copyable on mobile, delivered on the first response without a second request.
Commentary and analysis stay as normal prose **outside** the block.

This extends to **every ad-hoc hand-off** during or after a review — a profile link, a short
intro, a one-line availability note, a reworded answer. Deliver it in a copy-block on the first
response, **even for a single URL** (a bare clickable link still forces a manual select; the
block doesn't).

## First-draft voice

Apply the voice profile on the **first** draft — not as a correction after the user flags it.
Re-read the relevant knowledge-base slice and the voice profile before drafting if they haven't
been referenced recently in the conversation. An over-polished paragraph is itself the AI tell —
match the person's real rhythm, including its looseness.

## AI-tells to avoid (generic)

- No generic openers: "I am thrilled to apply…", "I am excited about the opportunity…".
- No corporate filler and no AI tropes: "delve", "testament", "moreover", "in today's
  fast-paced world".
- Avoid **em-dash-heavy sentences and stacked embedded clauses** — the single most common
  "this was written by an AI" signal. Prefer the person's real sentence shape.
- No emojis in application text.
- Name a technique or a metric plainly and move on; don't wrap it in a crafted line.

The profile's `voice-seed` lists the *person-specific* tells to avoid — apply those on top.

## Salary-ask heuristic (when asked for a number)

- **Posted range present:** anchor near the **top** of the range if the seniority genuinely
  supports it. The snapshot's floor is the safety check — **never ask below it.**
- **No posted range:** market-comparable for the role/region **+10–15%**, still floored at the
  snapshot's floor.
- **Floorless profile (no floor set):** don't apply a floor — estimate from the role/region with
  judgment and say so honestly.
- **B2B vs full-time:** state the number in the **basis the listing uses**; if B2B, it's the
  gross day/month rate the contractor model quotes, not a net salary.
- Give **one number or a tight range**, confidently, with a one-line justification — not a menu.

## Closing gate — on the listing's actual employment type

- **Explicitly freelance / contract / B2B listing:** if the profile's eligibility includes a
  contractor model, close by working the contractor advantage in naturally (fast integration,
  no payroll/visa overhead for their HR) — as the final paragraph, not bolted on. The
  `voice-seed` carries the profile-specific mechanism (e.g. a visa-bypass angle) when one
  applies.
- **Full-time (or ambiguous / unstated):** do **not** push the contractor framing. Close with a
  plain, direct availability/interest statement — full-time is a genuine option, never a
  fallback. Raise the contractor model only if it comes up naturally later.
- **Genuinely unclear which model applies:** ask the user once rather than guessing — this is the
  closing choice most likely to misfire if assumed.
